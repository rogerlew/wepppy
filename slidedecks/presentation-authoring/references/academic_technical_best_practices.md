# Academic / Technical Slide Best Practices

This reference captures evidence-backed rules for technical and academic talks.

## Core rules for primary slides
1. One major idea per slide.
- Source: Naegle (PLOS, 2021), Rule 1.

2. Target roughly one minute per slide.
- Source: Naegle (PLOS, 2021), Rule 2.
- Practical implication: if a slide needs more than a minute, split it.

3. Use assertion headings (message-first slide titles).
- Source: Naegle (PLOS, 2021), Rule 3.
- Example: prefer `Model error drops 32% with seasonal calibration` over `Results`.

4. Include only essential points and essential visuals.
- Source: Naegle (PLOS, 2021), Rules 4 and 6.
- Break multi-panel manuscript figures into staged, digestible slides.

5. Design to avoid cognitive overload.
- Source: Naegle (PLOS, 2021), Rule 7.
- Avoid full-sentence text blocks that compete with spoken narration.

6. Make each slide understandable even if attention drifts.
- Source: Naegle (PLOS, 2021), Rule 8.
- Litmus test: takeaway should be recoverable from heading + key visual.

7. Practice iteratively and time transitions.
- Sources: Bourne (PLOS, 2007), Rule 7; Naegle (PLOS, 2021), Rule 9.

8. Plan for technical failure modes.
- Sources: Naegle (PLOS, 2021), Rule 10; Bourne (PLOS, 2007), general readiness.

## Rapid/lightning format adjustments
1. Prepare one primary message for the whole talk.
- Source: Lortie (PLOS, 2017), Rule 1.

2. One major point per slide is non-negotiable in fast formats.
- Source: Lortie (PLOS, 2017), Rule 2.

3. Limit text and simplify visuals more aggressively.
- Source: Lortie (PLOS, 2017), Rules 3-4.

4. Repeat critical message once with a different visual.
- Source: Lortie (PLOS, 2017), Rule 6.

5. End with contact/follow-up resources and timed rehearsal.
- Source: Lortie (PLOS, 2017), Rules 9-10.

## Data-heavy research talks
1. Show only data needed for the story.
2. Prepare audience before dense figures.
3. Reveal complex figures in steps.
4. Keep backup slides for anticipated Q&A.
- Source: Grambow & Weinfurt (2024), sections on presenting data.

## Accessibility and inclusion signals
1. Verbally describe essential visual content; do not rely on visuals alone.
- Source: Peng et al. (arXiv 2021): many visual elements in real talks were insufficiently described.

2. Ensure projected content remains readable and high-contrast.
- Practical extension of cognitive-load and accessibility guidance across sources.

## Default conversion heuristic from long docs
- Keep about 10-20% of source text in primary slides.
- Keep supporting details in backup slides and notes.
- Preserve all factual claims, numbers, and caveats relevant to decisions.
