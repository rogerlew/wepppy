## Decisions D1–D5 (ratified by Roger, 2026-07-20)

- **D1 — Report render runtime**: **in-worker Quarto CLI.** QMDs are first-party trusted code; sidecar sandbox not needed.
- **D2 — PDF report**: **HTML-only at launch.** PDF/TinyTeX (or typst rework) deferred to follow-up.
- **D3 — Omni provisioning**: **user responsibility.** The user runs Omni scenarios and contrasts before configuring PATH-CE. The RQ task drops all auto-provisioning; it validates preconditions and fails with actionable "run Omni scenarios/contrasts" errors. Contrasts are structurally required (Sddc constraint columns only exist via outlet contrast deltas; solver raises if absent); users neutralize the constraint by setting `sddc_threshold` ≥ post-fire Sddc, not by omitting contrasts.
- **D4 — Treatment cost units**: **serialize $/acre in `path_ce.nodb`** (Jackson's native convention); UI displays both English and SI via unitizer. Vendored seam must compute cost vectors on an acre area basis (upstream multiplies $/acre against hectare areas — flagged to Jackson; ADR records our resolution).
- **D5 — Threshold sweep**: **keep, always run** with cache reuse (report section 3 sliders, 3D cost surface, and threshold plots depend on it). Parallelization/optimization is an explicit follow-up work package, out of scope here.

## Risks

- Users may launch PATH-CE against incomplete Omni artifacts (missing mulch contrasts, stale summaries after WEPP re-run); precondition validation and UI messaging must be specific about what to run.
- Threshold sweep is O(thousands) LP solves; unbounded ranges would stall RQ workers — keep upstream binary-search bounding + cache reuse; parallelization deferred to follow-up package (D5).
- Upstream area/cost unit inconsistency (ha × $/acre): our seam computes on acres per D4; parity tests vs upstream will show a deliberate cost-magnitude delta — document in ADR, confirm with Jackson.
- Inline HTML serving is a stored-XSS-adjacent surface; mitigations enumerated in package.md and verified in security review.
- Residual (platform-wide, accepted with rationale in the Phase 2 review disposition): RQ jobs execute the latest persisted config rather than an enqueue-time snapshot, and artifacts overwrite fixed paths non-atomically; the bp duplicate-run guard covers the realistic double-submit case. A platform-level initiative would be needed to change either property.
- Quarto adds image size/build time (HTML-only keeps this bounded per D2).
