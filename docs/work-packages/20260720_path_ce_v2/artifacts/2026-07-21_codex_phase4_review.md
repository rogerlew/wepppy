| # | Sev | Finding (condensed) | Disposition |
|---|-----|---------------------|-------------|
| 1 | High | Non-HTML active documents (scripted SVG/XHTML) served inline would execute in the service origin, outside the sandbox CSP. | **Accepted, fixed.** Inline serving is now allowlist-based (html/js/css/json/geojson/csv/png); anything else is forced to `application/octet-stream` + `Content-Disposition: attachment`. Regression tests serve scripted SVG and XHTML and assert download-only. |
| 2 | High | A symlinked `path/` or `path/report/` would be blessed as the containment root by `resolve()`, exposing a foreign tree. | **Accepted, fixed.** The fixed subtree components are symlink-checked before resolution; tests cover both a symlinked report root and a symlinked `path/` parent (403, foreign content never served). |
| 3 | Med | TOCTOU between containment checks and `FileResponse`'s later open; render service's rmtree-then-copy leaves a long partial-state window. | **Accepted in part.** Leaf race closed: files are opened with `O_NOFOLLOW`, validated via `fstat` (regular file), and served from the buffered read — a last-instant leaf symlink swap cannot serve outside content. Publication is now near-atomic (assemble beside the live tree, two renames, dot-prefixed staging names are unservable). **Declined**: full `openat`-walk descriptor serving and immutable versioned report directories — heavy machinery relative to the threat model (run trees are wepppy-managed; the residual is a directory-component swap in a microsecond window). Residual recorded for the Phase 5 security review. |
| 4 | Med | `allow-popups` provides an outbound navigation channel `connect-src` cannot govern (`window.open` with data in the URL). | **Accepted, fixed.** `allow-popups` removed (the report needs none); `base-uri 'none'` and `form-action 'none'` added. Live headless check confirms the report is unaffected. |
| 5 | Low | `private, max-age=60` lets authenticated report content persist in browser cache across logout/visibility changes. | **Accepted, fixed.** `private, no-store`. |
| 6 | High | Dynamic treatment inputs were never *registered* with the unitizer (`updateNumericFields` walks the registered set only), so canonical values could go stale on edit and new rows were unmanaged. | **Accepted, fixed.** `refreshUnitizer` now calls `registerNumericInputs` (which attaches the canonical-maintaining input handlers) before `updateNumericFields`; row-append refreshes too. |
| 7 | High | **Pre-existing platform defect surfaced by this package**: `ton→tonne` and `ton/yr→tonne/yr` conversions used factor **25.4** (the inch→mm factor) instead of 0.90718474 — an English-mode Sddc entry would inflate ~28× before reaching the solver. | **Accepted, fixed (platform fix).** `wepppy/nodb/unitizer.py` corrected for both `weight` and `weight-annual`; `unitizer_map.js` regenerated (the builder needs `write_unitizer_module()` — running the module as a script is a silent no-op, discovered en route); 9 factor/round-trip regression tests added (`tests/nodb/test_unitizer_conversions.py`). Recorded for Phase 5 parameterization evidence. |
| 8 | Med | Blank numeric fields silently became 0 (clearing Sdyd would reset the configured 15 to 0). | **Accepted in part.** Blank thresholds are now *omitted* from the payload so the server's partial-merge preserves the configured values (test added). Blank treatment quantity/cost/fixed-cost remain 0-valued — documented contract (the controller's normalization owns treatment validation); a fully-blank row is still skipped. |
| 9 | Med | `error_factory` bodies arrive as HTTP 200; the UI reported rejected saves as "Configuration saved" and rejected runs emitted `job:started`. | **Accepted in part.** The controller now detects error-shaped bodies on save and run, preserves the specific server message (handled-error tagging so the generic catch doesn't clobber it), emits error events, and never sets a job id. **Declined**: changing the blueprint to 4xx status codes — the 200-with-error-body shape is the platform-wide `error_factory` contract that other controllers rely on. Tests: invalid-save and active-run rejection end-to-end at the JS layer. |
| 10 | Low | Route tests mock authorization entirely; denial test accepted a range of status codes. | **Accepted in part.** The 403-denial assertion is now exact (403, no redirect — matching `handle_auth_error` semantics). **Declined**: standing up a real-token auth harness here — `authorize_run_request` is certified by its own suite (`test_browse_auth_routes.py`) and the report route calls it with the identical flags as `/download/` (parity by construction, asserted in review). |
| 11 | Low | Precondition display inferred from message prose; corrupt-artifact and watershed failures could miss the regex. | **Accepted, fixed.** Structured `precondition_errors` now flow controller → status endpoint → UI (stored on the nodb at failure, cleared at run start); the JS renders the structured list first and keeps a broadened prose fallback for older payloads. Tests at all three layers. |

## Notes

- Finding 7 is the package's most consequential platform catch: the 25.4 factor predates PATH-CE entirely and would have corrupted any English-display weight conversion flowing through the unitizer numeric-field machinery. Fixed at the source, regenerated, and pinned by tests.
- The Codex-recommended `registerNumericInputs` fix (finding 6) also retires the earlier UnitizerClient-presence heuristic's weakness: with registration in place, the canonical dataset is genuinely maintained on edit.
- Declines are platform-contract or threat-model calls, not disagreements on mechanism; each residual is listed for the Phase 5 security review (TOCTOU directory-component window; 200+error-body convention; auth-parity-by-construction).
