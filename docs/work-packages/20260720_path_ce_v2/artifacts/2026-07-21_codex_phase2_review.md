## Notes

- The finding-2 defect class is the same one Codex caught in Phase 1 (label↔column pairing); Phase 1 fixed the solver seam, this closes the config seam above it. The derivation now lives in exactly one place (`presets.label_for_scenario`).
- Cache-key semantics clarified during F5/F9 work: sub-integer threshold changes intentionally reuse the sweep (the grid is anchored on `int()` values and is unchanged); anchor-changing thresholds and any treatment/filter change invalidate. Now covered by tests.
- Declined portions of F1/F3 are platform-consistency calls, not disagreements about the mechanism: both are real properties of the wepppy RQ architecture and belong to a platform-level initiative if Roger wants them addressed; recording here keeps the residual risk visible.
