# Browse Arrow-to-Pandas Inventory

**Date**: 2026-06-16 19:10 UTC  
**Scope**: `wepppy/microservices/browse` and `wepppy/microservices/parquet_filters.py`

## Initial Call Sites

The package inventory found these production browse request-path conversions:

- `wepppy/microservices/browse/_download.py`
  - imported pandas
  - converted parquet paths/file handles into pandas DataFrames
  - called `table.to_pandas()` before CSV export
- `wepppy/microservices/browse/flow.py`
  - called `env.pd.read_parquet(...)` for unfiltered parquet previews
  - rendered filtered parquet previews through `query_preview(...).df()` and pandas HTML conversion
- `wepppy/microservices/parquet_filters.py`
  - returned pandas DataFrames from `query_preview(...)` through DuckDB `.df()`

The D-Tale browse bridge at `wepppy/microservices/browse/dtale.py` only forwards path and optional `pqf` metadata to the separate D-Tale service. It does not convert parquet to pandas in `browse`.

## Replacement Decisions

- Added `wepppy/microservices/browse/parquet_tables.py` for Arrow-backed projection, pandas physical index column dropping, unit-label handling, HTML rendering, CSV writing, and RSS telemetry helpers.
- Changed `wepppy/microservices/parquet_filters.py::query_preview` to return a PyArrow table instead of a pandas DataFrame.
- Changed parquet previews in `wepppy/microservices/browse/flow.py` to use bounded DuckDB/PyArrow preview tables and direct Arrow-to-HTML rendering.
- Changed parquet-to-CSV download conversion in `wepppy/microservices/browse/_download.py` to avoid DataFrames:
  - unfiltered CSV uses `pyarrow.parquet.ParquetFile.iter_batches(...)`
  - filtered CSV uses the existing filtered Arrow table export after the row cap is enforced
  - CSV header projection drops pandas physical index columns and appends units metadata like the previous DataFrame path
- Kept pandas in `browse` for non-parquet preview paths:
  - pickle preview
  - CSV preview
  - TSV preview

## Behavioral Notes

- Parquet HTML previews are now capped by `BROWSE_PARQUET_PREVIEW_LIMIT` for both filtered and unfiltered previews. This is an intentional hardening change to avoid rendering entire large parquet files into a long-lived worker.
- CSV formatting is generated by Python's standard `csv.writer` over Arrow rows. Headers and units/index-column behavior are covered by `tests/microservices/test_download.py`.
- Filter semantics remain owned by `docs/schemas/weppcloud-browse-parquet-filter-contract.md` and the existing DuckDB filter compiler.
