  Evidence: `artifacts/apprehensive_caw_timing_profile.md`, `artifacts/apprehensive_caw_timing_raw.json`.
- Observation: Existing lane candidate ordering is already strongly aligned to this timing profile: duplicate rebuild consolidation first, then logging compaction, then guarded pair-count reuse.
  Evidence: `artifacts/landuse_disturbed_pipeline_optimization_candidates.md`.
- Observation: Investigation scope originally did not require dedicated security artifact, but execution scope now warrants one due to mutation-sequencing and run-state integrity concerns in controller hot paths.
  Evidence: package scope change request at 2026-04-24 23:41 UTC.
- Observation: Full-run parity harnessing on cloned `apprehensive-caw` data was prohibitively expensive for iterative lane-mode comparisons during this turn; a deterministic isolated-temp lane harness was used to validate contracts and deltas without mutating source run trees.
  Evidence: `notes/run_landuse_disturbed_pipeline_lane_benchmark.py`, `artifacts/lane_benchmark_raw.json`, `artifacts/lane_parity_raw.json`.
