## Decisions Log

### 2026-04-28 20:52 UTC: Treat storm shape as a closed enum, not free text

**Context**: The requested UI exposes six named shapes. Existing Geneva validation already rejects unsupported distribution IDs.

**Options considered**:
1. Keep `distribution_type` as arbitrary string and let Rust reject unsupported values.
2. Define a shared closed enum across UI, Python validation, query/report payloads, and Rust kernels.

**Decision**: Option 2.

**Impact**: This preserves strict input validation, keeps report artifacts predictable, and avoids a new arbitrary-user-input surface.

---

### 2026-04-28 20:52 UTC: Default missing storm shape to NEH-4 B

**Context**: Existing schema defaults and UI payloads use `neh4_type_b`, even though the Python execution path currently builds uniform rainfall.

**Options considered**:
1. Default missing values to `uniform` to match the old runtime artifact behavior.
2. Keep the public/default contract at `neh4_type_b` and fix runtime behavior to honor it.

**Decision**: Option 2, unless a product owner explicitly approves a default change during implementation.

**Impact**: New and missing-value payloads retain the intended/public Type B contract. The implementation must still document that old completed artifacts may have been generated by the interim uniform Python path.
